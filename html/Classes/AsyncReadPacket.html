AsyncReadPacket Class Reference

Inherits from NSObject
Declared in AsyncSocket.m

Overview

The AsyncReadPacket encompasses the instructions for any given read. The content of a read packet allows the code to determine if we’re: - reading to a certain length - reading to a certain separator - or simply reading the first chunk of available data

– searchForTermAfterPreBuffering:

For read packets with a set terminator, scans the packet buffer for the term. It is assumed the terminator had not been fully read prior to the new bytes.

- (NSInteger)searchForTermAfterPreBuffering:(NSUInteger)numBytes

Discussion

For read packets with a set terminator, scans the packet buffer for the term. It is assumed the terminator had not been fully read prior to the new bytes.

If the term is found, the number of excess bytes after the term are returned. If the term is not found, this method will return -1.

Note: A return value of zero means the term was found at the very end.
